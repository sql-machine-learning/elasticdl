ElasticDL is a Kubernetes-native deep learning framework built on top of TensorFlow 2.0 that supports fault-tolerance and elastic scheduling.

Elastic Scheduling and Fault-Tolerance on Kubernetes

Through Kubernetes-native design, ElasticDL enables fault-tolerance and works with the priority-based preemption of Kubernetes to achieve elastic scheduling for deep learning tasks.

Support for TensorFlow 2.0

ElasticDL is implemented on top of TensorFlow 2.0 API that focuses on simplicity and ease of use with support for eager execution and intuitive high-level model building APIs.

Simple but Flexible High-level API

ElasticDL enables model building through Keras API and provides simple CLI to submit distributed deep learning tasks on Kubernetes with fault-tolerance and elastic scheduling.

                    
elasticdl train \
--model_zoo=model_zoo \
--model_def=mnist_functional_api.custom_model \
--training_data_dir=/data/mnist/train \
--evaluation_data_dir=/data/mnist/test \
--num_epochs=2 \
--minibatch_size=64 \
--records_per_task=100 \
--num_workers=2 \
--grads_to_wait=2 \
--job_name=test-mnist-train \
--output=elasticdl-saved-model

Integration with SQLFlow

ElasticDL will be integrated seamlessly with SQLFlow to connect SQL to distributed deep learning tasks with ElasticDL.

                    
SELECT * FROM employee LABEL income INTO my_elasticdl_model
